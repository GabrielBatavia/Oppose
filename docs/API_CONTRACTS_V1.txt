OPPOSE API CONTRACTS V1
Date: 2026-05-19

Purpose:
Draft backend API contract for moving Oppose from local mock state to real backend services. Sprint 13 implements the system endpoints plus auth/profile endpoints using development auth.

Base URL local:
http://localhost:4000

Authentication:
- Sprint 13 private endpoints require development-only header `x-dev-user-id`.
- Seeded local dev user ID: `00000000-0000-4000-8000-000000000001`.
- Future production auth should replace dev auth with Authorization: Bearer <token>.
- Production backend must derive current user from the token, not from client-provided user IDs.

Response format recommendation:
- Success: JSON object or array as endpoint requires.
- Error: { "error": { "code": string, "message": string, "details"?: object } }

AUTH AND PROFILE
================================================================================

POST /auth/session
Purpose: exchange provider token/session for backend user context.
Request:
{
  "provider": "supabase|firebase|clerk|custom",
  "providerToken": "string"
}
Response:
{
  "user": UserDto,
  "isNewUser": true
}

GET /me
Purpose: get current user profile.
Auth: x-dev-user-id in Sprint 13.
Response: UserDto

PATCH /me/profile
Purpose: update profile identity.
Auth: x-dev-user-id in Sprint 13.
Request:
{
  "displayName": "Bima's Friend",
  "username": "thinkwithbima",
  "avatarUrl": "https://...",
  "language": "en"
}
Response: UserDto

PATCH /me/interests
Purpose: replace current user interests.
Auth: x-dev-user-id in Sprint 13.
Request:
{
  "interests": ["Technology", "Friendship", "Food"]
}
Response: UserDto

POST /me/ai-consent
Purpose: record AI consent acceptance.
Auth: x-dev-user-id in Sprint 13.
Request:
{
  "accepted": true,
  "version": "2026-05-19"
}
Response: UserDto

GET /users/check-username?username=thinkwithbima
Purpose: username availability check.

Response:
{

  "available": true
}
