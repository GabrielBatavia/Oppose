===============================================================================
2. PRODUCT SUMMARY
===============================================================================

Product name:
Oppose

Core brand line:
Different take, better talk.

Product category:
Daily social AI communication app for friends, small groups, casual debate, and respectful disagreement.

Oppose is:
- a daily social app
- a friendly debate app
- a group chat and voice room app
- an AI-assisted conversation app
- a warm, culturally distinctive, scrapbook-style mobile experience
- a product that makes disagreement feel safe, friendly, and useful

Oppose is not:
- a political warfare app
- a gambling app
- a crypto/token app
- a cold AI assistant app
- a corporate productivity dashboard
- a generic chat clone
- a futuristic sci-fi control panel

Core user loop:
1. User signs up.
2. User creates a profile identity.
3. User selects interests.
4. User understands AI consent.
5. User lands on Home.
6. User chats with friends or creates a room.
7. User joins a voice room.
8. User can ask AI for help.
9. User can leave safely.
10. User receives an optional summary.
11. User returns later for another daily debate.

===============================================================================
7. CORE UX PRINCIPLES
===============================================================================

1. Comfort before complexity.
Every main action should be understandable within 3 seconds.

2. Build a real app, not an art board.
The UI must be component-based, responsive, and interactive.

3. AI with consent.
Users must always know when AI is off, listening, thinking, speaking, summarizing, or saving.

4. Safety controls must be visible.
Users must be able to leave, mute, report, block, and turn off AI.

5. Voice rooms must feel stable.
Even if the first implementation uses mock state, design the architecture for real WebRTC later.

6. AI must not dominate.
Default AI mode is Quiet Helper. AI answers only when asked or explicitly triggered.

7. Summaries are optional and deletable.
Never imply automatic hidden capture.

8. Social-first, debate-second.
Oppose is about better conversation with friends, not public conflict.

9. Native text only.
Reference images guide design; coded app should use native text components.

10. Accessibility is not optional.
Status cannot rely on color alone. Use labels, icons, and semantic roles.

===============================================================================
13. AI SAFETY AND CONSENT RULES
===============================================================================

AI must always be permission-based and visible.

Required AI states:
- AI Off
- Not listening yet
- AI Listening
- AI Thinking
- AI Speaking
- Summarizing
- Memory Off

Required AI controls:
- Ask AI
- Turn off AI
- Change AI settings
- Choose AI mode
- Choose summary setting
- Delete summary

Required copy:
- “AI only listens when it is turned on.”
- “You can mute or remove AI anytime.”
- “Summaries are optional and deletable.”
- “Generated by AI from allowed room context.”
- “Memory is off.”
- “AI will not save this conversation for future chats.”

Never:
- imply hidden listening
- show AI as a normal human user
- save summaries without a visible privacy state
- hide the AI off button
- make AI interrupt by default
- store private transcripts in analytics
- create long-term memory without explicit user opt-in

AI modes:
- Off: no AI in room
- Quiet Helper: responds only when asked
- Brainstormer: suggests ideas when prompted
- Translator: translates when requested
- Moderator-lite: helps keep discussion respectful when selected

Default mode:
Quiet Helper, or Off if the user has not accepted AI consent.

- Main flow tested.
- Edge states tested.
- Small screen tested.
